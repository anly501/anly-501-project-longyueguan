Preparing to start the Project, next step: Data Cleaning



This section is a data cleaning section, and the purpose of this section is to prepare our data for further analysis and predictions. Data cleaning is cleaning data to fix errors, delete duplicates and remove irrelevant data. And in our case, we also did the process to tokenize the text columns. The benefit of data cleaning is to stay organized because if we get tidy data, we can use it more effectively. And it will also help us to avoid mistakes and make our data and next step more accurate. Sometimes, the data cleaning section could be easy, and it could be simply removing empty values and changing column names. However, some could be complicated, such as creating a new column with tokenized text data and having more steps or creating more columns to get ready.

Specifically, we read the raw data and save it into DataFrame type. Then we replace all empty values with NA, get unique data, and remove emoji from the text column. That is what we did for the first part of data cleaning; however, after we did all methods, we improved more steps in data cleaning. We also did remove the URL, remove stop words, remove punctuation and remove words containing numbers. Also, since we are doing sentiment analysis, we also created a new column for finding positive and negative comments. And the column will return all positive and negative words exits in the text of the tweets. The last step is to count and vectorize the data and make it into the data frame column.
