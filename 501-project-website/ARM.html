ARM and Networking:

ARM is Association rule mining learning, which is an unsupervised rule-based machine learning method for discovering statistically relevant inter-relations between variables in large databases. And Network Analysis is always used in application tasks. And it could help us quickly understand the structure of the relationship between different values. For example, network analysis could help us find the relationship between users and followers on Twitter.

This section provides the usage of ARM. Since the ARM is a learning technique to identify the dependencies between two data groups. We want to use it in our sentiment analysis based on climate change from tweets. We will try to find three groups of transaction data in the ARM process. And the first group would be the mixed positive and negative words from tweets data, the second group would be only positive words, and the last group should be only negative words. And then, we would like to compare those results and find beneficial effects.

Then we will use our tweets data; however, it is a different process from what we did from previous methods, such as Naive Bayes, Decision Trees, and SVM. We first would like to re-clean the data from the firstly API dataset. We did a clean process again to get the expected nested list, which is a transaction data to work on the process, which is getting ARM and also plotting network objects. To make all methods work correctly, we must transfer all data from the dataframe to a nested list and match their position. After doing all these, our data is ready to use and will get the results.

We did use ARM in this part, and it will provide us support, confidence, support*confidence, and lift values. And support threshold value measures the percentage of all item-set A and B together and compared with all other transactions. Confidence measures the percentage of A and B items that occur when they contain A. Also, When A and B are independent, the lift could calculate the percentage of observed versus expected. Since we also have support*confidence, it could provide the metrics of the product of support and confidence.

And overall, we used the Apriori Algorithm of ARM in the process. And Apriori Algorithm is the method that could calculate rules that express the probabilistic relationship between items. The main idea of the Apriori Algorithm is all non-empty subsets of a frequent itemset must also be frequent. According to the Apriori Algorithm, we could notice that the function of getting the support threshold is equal to the number of transactions containing both X and Y divided by the total number of transactions. And the confidence equals the number of transactions containing both X and Y divided by the number of transactions only containing X. And in our case, when we run the Apriori Algorithm, we set up the min_support as 0.003, min_confidence as 0.02, min_length as 1 and max_length as 5.

We also use network analysis to help us quickly understand the relationship with those transaction data. As we know, network graphs are structured with nodes and edges. And it has two types of edges, which are un-directed and directed. In this case, we have directed edges on all network graphs. And all edges can correspond to different interactions. And we are going to use it to find out how it interacts with different sentiment words on climate change topic.

And to be more specific, edges could represent the presence of a relation among the elements defined by the nodes. And nodes are a set of elements. And directed edges will represent an arrow between the nodes. And if the arrow is from A to B, then A has a direct relationship with B. And in our case, since we are doing sentiment analysis, our nodes would be sentiment words, and we could see if they have a relationship with each other with directed edges. If they have a direct connection, they have a close relationship. It could also have a situation that the relationship is connected with two directed edges. Then that means not closer than the previous case.
